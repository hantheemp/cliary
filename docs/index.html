cliary_

The terminal journal that runs entirely on your machine.

Type fragments whenever something happens. cliary merges them into one honest daily entry using a local language model, and never sends a single word anywhere else.

$ cliary add "finished the auth flow, felt good"
[OK] Entry saved.
$ cliary correct
[INFO] Processing 1 day(s)... [OK] Correction completed.
$ cliary view
# 2026-07-12 I finished the auth flow today, it felt good.
Get Started → View source
git clone https://github.com/hantheemp/cliary.git && cd cliary && npm install && npm run build && npm link

Requires Node.js 22 LTS or newer

why cliary

Most journaling tools expect you to sit down and write everything at once.

  • Capture a thought the moment it happens, in a terminal that's probably already open.
  • No formatting, no editing, no pressure to make it sound right while you type.
  • One command merges the day's notes into a coherent, corrected entry.
  • Your original notes are always kept, unedited, right alongside it.
local-first, by design

Nothing you write ever leaves your machine.

Local SQLiteEvery entry lives in one file on disk: ~/.cliary/cliary.db
Local LLMCorrection runs through Ollama or any OpenAI-compatible server on your own machine
No cloudNothing is synced or uploaded anywhere by cliary itself
No accountNothing to sign up for, nothing to log into
No telemetrycliary doesn't phone home, ever

The LLM is entirely optional. Every command works with no model configured; only cliary correct needs one, and only when you choose to run it.

see it work

Your typos included. That's the point.

Three rushed, lowercase notes, typed exactly as you'd type them mid-day, become one clean entry.

[09:15]went to the offise, there was some trafic today

[13:00]had lunch with my collegues, it was nise

I went to the office, there was some traffic today. I had lunch with my colleagues, it was nice. I'm really tired now, going to sleep.
